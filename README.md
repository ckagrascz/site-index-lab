# site-index-lab

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

site-index-lab 是一个轻量级的 HTML 页面存档仓库，主要用于存放各类独立开发的网页文件。这些页面可能是一些实验性作品、静态展示页面、小型工具页面或其他无需后端支持的 HTML 资源。本仓库不针对任何单一域名或具体网站，仅作为集中存储与版本管理之用。

## 目录结构说明

仓库根目录下包含以下主要内容：

- `pages/` — 存放所有独立 HTML 页面文件，每个页面可位于子目录中，便于按主题或用途分类。
- `assets/` — 页面所需的共用资源，如样式表、脚本、图片或字体文件。
- `index.html` — 仓库的入口页面，提供对 `pages/` 内各页面的简单导航链接（可选）。
- `README.md` — 即本文件，用于说明仓库用途与使用方式。

您可以根据需要自行调整目录结构，但请保持基本的组织清晰。

## 页面归档说明

所有页面均为独立的 HTML 文件，不依赖外部动态服务或数据库。每个页面应包含完整的 HTML 结构，并尽量自包含（内联样式与脚本），以确保在离线或静态托管环境下正常工作。

若页面使用了外部资源（如 CDN 库），请在文件头部注释说明来源及版本，便于后续维护。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 下对应分类目录，并确保所有资源路径正确。
- 更新页面：直接修改对应文件，提交时请在 commit 信息中简要描述改动内容，例如“更新页面样式”或“修复链接错误”。
- 删除页面：移除对应文件及目录，并清理 `assets/` 中不再使用的资源（如有）。
- 不建议在页面中添加任何形式的跟踪代码、营销内容或诱导性跳转链接，保持页面的纯粹与透明。

## 使用方式

您可以通过以下方式查看仓库中的页面：

1. 克隆仓库到本地，在浏览器中直接打开 `pages/` 下的 HTML 文件。
2. 将仓库部署到静态托管平台（如 GitHub Pages、Netlify 等），通过对应链接访问页面。

本仓库不提供任何形式的访问统计、广告投放或用户数据收集功能。
